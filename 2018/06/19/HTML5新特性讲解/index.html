HTML5新特性讲解

发表于 2018-06-19 | 分类于 HTM5

HTML:超文本标记语言(页面中不仅只有文字，而且可以呈现出图片、音视频等媒体资源)
XHTML:它是HTML比较规范严谨的一代版本
XML: 可扩展的标记语言（HTML中使用的标签都是W3C标准中规定的，XML允许我们自己扩展标签的），它的作用不是用来写页面结构的，而是用来存储一些数据的（以自己扩展的标签作为标识，清晰明了的展示出数据结构…）
ajax: async javascript and xml
HTML5：当前HTML最新的一代版本，也是非常成功的一代版本，目前市场上都是基于H5规范进行开发的，它相对于传统的HTML更多的是增加一些有助于开发的内容，对原有规范的修改调整很少

HTML5提供的新语法规范

对原有语义化标签的升级

语义化标签（标签语义化）：每一个HTML标签都有自己特殊的含义，我们在搭建页面结构的时候，应该让合理的标签做合适的事情
HTML5中新增加一些语义化标签：

H5中新增加一些标记标签

H5中对原有标签还有一些调整

H5中删除一些不经常使用的标签：这里的删除不是不让用（用了也不会报错），只是按照最新标准没有语义了

目前不管是在PC端开发还是移动端开发，我们更应该使用H5规范的语义化标签搭建页面的结构
问题：
IE6~8中不能识别这些新增加的语义化标签，我们无法为其设置具体的样式：结局方案：在当前页面的HEAD中(css后)，我们导入一个JS插件：html5.min.js，它就是用来把页面中所有用到的不兼容的H5语义化标签进行兼容处理
1,把页面中所有不兼容的标签进行替换
2,把css中使用标签选择器设置的样式（标签是H5标签）也替换成其它方式，但是这样比较耗性能，所以我们最好自己写样式类
标准浏览器中不需要引用，只有ie6~8中使用(使用条件注释来区分浏览器)

H5中对于表单元素的升级

传统表单元素
form input text password radio checkbox button submit reset file hidden
button
select
label
textarea
submit点击按钮会跳转到form的action对应的地址（表单提交）,在传统的非前后端分离的项目中，我们会在ACTION中指定一个程序处理页面（一般是由后台语言（PHP\JSP\ASP…）完成），我们利用它的默认行为把数据发送给处理页面，由处理页面完成数据的存储等操作。
现有前后端完全分离项目中，我们都是在js中手动获取到用户输入的内容，并且通过ajax等技术发送给服务器存储或者处理（此时我们需要阻止submit默认行为）
…
单选框需要name进行分组，一组中只能选中一个,label有一个for属性，指向对应表单元素的ID，操作LABEL中的内容，也相当于在操作表单元素（经常用于单选或者复选框上）
H5对于表单的升级
1、给input设置了很多新的类型
search
email
tel
number
rangle min max step value
color 会调取颜色面板
date
time
..
优势：1、功能强大了 2、使用合适的类型，在移动端开发的时候，用户输入可以调取出最符合内容格式的虚拟键盘，方便用户操作
部分类型提供了表单验证（内置验证机制）

2、新增加了placeholder,给表单框做默认的信息提示
3、二级下拉框(select一级下拉框) 比起select多了一个你输入关键字到你的选项中自动筛选的功能，很强大

需要注意的是以上HTML5对表单的升级，对ie的兼容性都不是很好，有的ie9和ie10都不兼容，而且没办法处理兼容，所以我们一般在移动端使用这些HTML标签，pc端还是延续传统的操作办法来操作
H5中的表单验证内置规则不是特别好，所以真实项目中的表单验证，依然延续传统的正则验证完成

解决placeholder的兼容处理(新的一种组件封装思想)

整个ie浏览器对placeholder兼容性都不好，ie10及以上，虽然兼容，但是文本框获取焦点后，提示信息就消失了
ie9及以下不兼容这个属性

H5中其它新增内容

增加了新的媒体解决方案
音频：audio
视频：video
传统的音视频播放是基于flash来完成的，需要浏览器中安装adobe flash player插件
现在只需要基于audio或者video播放即可，但是对于音视频的格式有限制，对于浏览器也有限制
移动端对于flash的支持不好，但基本上都支持audio和video
pc端的ie浏览器（低版本）不支持audio和video,但是支持flash
一般如果在移动端我们就用audio和vedio就可以了，在pc端一般我们用人家已经开发好的视频播放器(例如，腾讯，爱奇异)

H5中增加了canvas绘图，它是一个画布，允许我们在js中通过代码绘制图形以及实现一些好玩的动画，百度统计图插件:Echarts就是基于canvas开发的
提供了很多强大的JS API
API:凡是提供一个供别人调取使用的，都可以称之为接口，例如：使用AJAX从服务器端获取数据，需要一个URL地址，这个地址就是一个API，浏览器提供给我们很多常用的方法，每一个方法都可以叫做API，API是一个泛指

本地存储

webStorage:分为两大类：
localStorage:本地信息存储
sectionStorage:本地会话存储
在没有H5存储之前，我们都是用cookie做的本地存储
缓存机制和本地存储是两个机制，缓存大多存在虚拟内存中，叫cache，是浏览器的机制

获取本机地理位置

通过H5可以获取当前用户地理位置（精度、维度、精准度…），再结合第三方地图（高德地图、百度地图、腾讯地图…）API接口，实现一些生活服务的推荐等。

提供了新的通讯方式：websocket

想要实现实时通讯类的产品，基本上现在都是基于socket.io这个框架来完成的

提供了操作手机硬件功能的API

调取手机的重力感应器，实现摇一摇，或者实现一些小游戏
调取手机的摄像头或者通讯录等
不是所有的手机浏览器都支持这些功能，即使支持这些功能的浏览器，在实现效果上也是不理想的（不稳定、卡顿等）

H5离线缓存:manifest

第一次联网请求完成页面，把信息缓存到本地，下一次即使断网的情况下，也可以看到上一次的信息
